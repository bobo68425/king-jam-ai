# 一、執行摘要

## 1.1 平台定位

**King Jam AI** 是一個一站式 AI 內容生成與社群管理平台，專為創作者、企業和行銷人員設計，幫助用戶快速產出高品質內容並自動化社群經營。

## 1.2 核心價值主張

> **「讓 AI 成為每個人的創作夥伴」**

透過專有的「三明治 Prompt 架構」，將專業級 AI 內容生成能力民主化，讓沒有技術背景的用戶也能產出電影級品質的短影音內容。

## 1.3 平台規模一覽

| 指標 | 數值 | 說明 |
|:-----|-----:|:-----|
| API 端點數 | **123+** | RESTful API |
| 資料庫表數 | **21** | PostgreSQL |
| 前端頁面數 | **15** | Dashboard 頁面 |
| Docker 服務數 | **8** | 微服務架構 |
| Router 模組 | **27** | 業務模組 |
| Service 模組 | **18** | 服務層 |

## 1.4 關鍵亮點

- ✅ **多模型容錯機制** - Veo → Kling → Imagen 自動降級
- ✅ **三明治 Prompt 架構** - 用戶簡單輸入 → 專業級輸出
- ✅ **四層點數系統** - 靈活的貨幣化策略
- ✅ **8+ 社群平台整合** - 一鍵發布多平台
- ✅ **推薦分潤機制** - 10%-20% 分潤驅動增長

## 3.3 微服務隔離策略

| 服務類型 | Redis 實例 | 佇列名稱 | 原因 |
|:---------|:-----------|:---------|:-----|
| 核心服務 | redis:6379 | queue_high | 確保驗證碼、登入即時性 |
| 一般服務 | redis:6379 | queue_default | 排程、發布任務 |
| 影片服務 | redis-video:6380 | queue_video | 防止 OOM 影響核心 |

## 3.4 資料庫連接池配置

```python
# 連接池參數
POOL_SIZE = 20          # 基本連接數
MAX_OVERFLOW = 30       # 最大溢出連接
POOL_TIMEOUT = 30       # 等待超時 (秒)
POOL_RECYCLE = 1800     # 連接回收時間 (秒)
POOL_PRE_PING = True    # 連接健康檢查
```

**優化效果：**
- 支援 400+ 併發請求
- 平均響應時間 < 100ms
- 最高響應時間 < 600ms (200 併發)

### 4.1.2 場景類型定義

| 場景類型 | 英文代碼 | 用途 | 建議時長 |
|:---------|:---------|:-----|:---------|
| 開場 | OPENING | 吸引注意力 | 1-2 秒 |
| 展示 | SHOWCASE | 展示產品/服務 | 2-3 秒 |
| 特寫 | CLOSEUP | 細節呈現 | 1-2 秒 |
| 轉場 | TRANSITION | 場景過渡 | 0.5-1 秒 |
| 高潮 | CLIMAX | 情緒最高點 | 2-3 秒 |
| 結尾 | ENDING | CTA 呼籲行動 | 1-2 秒 |

### 4.1.3 旁白強制生成機制

```python
# 創作原則（系統提示詞）
"必須有旁白：每個場景都必須包含 narration_text（旁白文字），不可留空！"

# Fallback 機制
if not narration_text:
    narration_text = self._get_fallback_narration(scene_type, brand)

# 品牌名稱智慧處理
if brand_name in ["我的品牌", "My Brand", "Brand", ""]:
    # 使用「我們」或通用稱謂代替品牌名稱
    narration_text = narration_text.replace("我的品牌", "我們")
```
